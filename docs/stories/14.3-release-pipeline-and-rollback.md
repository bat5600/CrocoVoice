## Story
**As a** release manager,
**I want** a repeatable release pipeline with rollback,
**so that** updates are safe, auditable, and recoverable.

## Acceptance Criteria
1. CI pipeline builds signed Windows artifacts and publishes them to a release location.
2. Update feed/manifest is generated per release with version + checksum.
3. Release notes and versioning policy are documented.
4. Rollback runbook is documented and validated (latest-to-previous release).
5. Manual update fallback path is documented for users.
