## Story
**As a** product owner,
**I want** a Windows-first packaging and signing pipeline,
**so that** CrocoVoice can be installed and trusted by external users.

## Acceptance Criteria
1. Packaging tool is selected and documented (with rationale and constraints).
2. Windows installer artifacts are produced (exe/msi) with consistent versioning.
3. App is signed with a valid Windows certificate (or documented interim workflow if unavailable).
4. Build configuration includes app identity (name, id, icon, version, publisher).
5. Packaging output is reproducible via a single command (local) and CI-ready.
6. No regressions in core app launch and recording flow on Windows.
